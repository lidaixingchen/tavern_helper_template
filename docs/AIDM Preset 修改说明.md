# AIDM Preset 预设修改说明文档

## 一、参考文件分析结果

### 1.1 Kemini Aether final.json 结构分析

参考文件是一个 SillyTavern 预设文件，包含以下主要结构：

#### 参数配置
| 参数 | 原值 | 说明 |
|------|------|------|
| temperature | 1.25 | 较高的创造性温度 |
| top_p | 0.98 | 高核采样率 |
| top_k | 500 | 大词汇范围 |
| openai_max_tokens | 65535 | 最大输出token |

#### Prompts 结构特点
- 共包含约30个提示词模块
- 使用emoji图标命名（如🧭、⚓️、➡️、🎁等）
- 采用分层注入系统（injection_position, injection_depth, injection_order）
- 包含大量正则表达式处理（clewd正则）
- 核心模块包括：
  - 系统设定（角色扮演身份定义）
  - 主提示（创作风格指导）
  - 思维链（CoT思考框架）
  - 去八股（反AI写作风格）
  - NSFW风格（成人内容处理）
  - 情绪优化（角色情感表达）

### 1.2 参考文件风格特征

1. **互动式小说导向**：专为角色扮演和互动叙事设计
2. **复杂正则处理**：包含大量文本替换和格式化规则
3. **思维链机制**：要求AI在输出前进行结构化思考
4. **反AI写作**：通过"去八股"模块避免AI痕迹
5. **详细风格指导**：包含fiction_style、sex_style等详细写作规范

## 二、世界书风格分析

### 2.1 世界书组织结构

世界书采用模块化YAML格式，按功能分类：

```
00-核心定义/    → DM原则、SPV体系、文风指南
01-数值参考/    → 战斗、技能、经济、装备数值
02-角色系统/    → 属性、技能、装备规则
03-战斗系统/    → 战斗流程、伤害计算、状态效果
04-检定系统/    → 标准检定、对抗检定、DC表
05-世界系统/    → 世界设定、乾坤界、剧情系统
06-事件系统/    → 触发条件、输出模板
07-变量定义/    → 变量初始化、更新规则、输出格式
08-示例与模板/  → 战斗示例、检定示例、输出模板
```

### 2.2 世界书风格特征

1. **规则至上**：DM核心原则强调公平性和规则遵守
2. **数值驱动**：所有判定基于D20骰子+属性加成
3. **结构化输出**：使用固定模板（剧情模式、战斗模式、检定结果）
4. **按需输出**：数据表仅在必要时显示，常规回合保持叙事流畅
5. **简洁高效**：场景描述字数有明确限制，避免冗余

## 三、具体修改点

### 3.1 参数配置修改

| 参数 | 原值 | 新值 | 修改理由 |
|------|------|------|----------|
| temperature | 1.25 | 1.0 | 降低随机性，确保规则判定的一致性 |
| top_p | 0.98 | 0.95 | 收窄采样范围，提高输出稳定性 |
| top_k | 500 | 100 | 减少词汇选择范围，避免偏离规则 |
| min_p | 0 | 0.05 | 添加最小概率阈值，过滤低质量输出 |
| repetition_penalty | 1 | 1.05 | 轻微增加重复惩罚，避免循环描述 |
| openai_max_tokens | 65535 | 4096 | 限制输出长度，符合300-500字要求 |

### 3.2 Prompts 模块重构

#### 删除的模块
| 模块名 | 删除理由 |
|--------|----------|
| clewd正则（复杂版） | 原正则过于复杂，包含大量特定场景处理，不适合跑团 |
| 去八股 | 跑团需要规则化输出，不需要文学化处理 |
| NSFW风格 | 跑团系统不涉及成人内容 |
| 情绪优化 | 跑团强调规则判定，不需要情感渲染 |
| 防截断 | 跑团输出有固定长度限制，不需要额外填充 |

#### 保留并简化的正则模块
| 模块名 | 修改内容 | 功能说明 |
|--------|----------|----------|
| 格式正则 | 简化为跑团专用格式美化 | 美化思维链和输出格式 |

## 四、修改理由和依据

### 4.1 参数调整依据

1. **降低temperature**：跑团游戏需要一致的规则判定，过高的temperature会导致骰子结果和规则应用不稳定

2. **收窄采样范围**：跑团输出需要严格遵循模板格式，过宽的采样范围可能导致格式偏离

3. **限制输出长度**：世界书明确要求普通场景100-150字，战斗场景80-120字，总输出300-500字

### 4.2 模块重构依据

1. **删除文学化模块**：世界书强调"传递关键信息为主，展示文采为辅"，不需要去八股、情绪优化等文学处理模块

2. **新增规则模块**：跑团游戏的核心是规则系统，需要明确注入检定系统、战斗系统等规则

3. **标准化输出**：世界书定义了严格的输出模板，需要确保AI遵循这些格式

### 4.3 思维链重构依据

原思维链关注：
- 剧情发展
- 角色情感
- 写作风格

新思维链关注：
- 场景分析（时间、地点、人物位置）
- 玩家意图解读
- 规则判定（是否需要检定、DC值、投骰结果）
- 输出规划（选择合适模板）

## 五、新旧预设对比说明

### 5.1 设计理念对比

| 维度 | Kemini Aether | AIDM Preset |
|------|---------------|-------------|
| 目标场景 | 互动式小说 | AI无限流跑团 |
| 核心机制 | 角色扮演+情感互动 | 规则判定+骰子投掷 |
| 输出风格 | 文学化、情感化 | 结构化、数值化 |
| 灵活性 | 高（自由创作） | 中（遵循规则） |
| 复杂度 | 高（多模块协同） | 中（规则驱动） |

### 5.2 模块数量对比

| 类型 | Kemini Aether | AIDM Preset |
|------|---------------|-------------|
| 总模块数 | ~30 | 19 |
| 系统模块 | 5 | 3 |
| 规则模块 | 0 | 6 |
| 格式模块 | 3 | 4 |
| 正则模块 | 1（复杂） | 1（简化） |
| 辅助模块 | 21 | 5 |

### 5.3 功能兼容性

| 功能 | 兼容性 | 说明 |
|------|--------|------|
| 世界书注入 | ✅ 完全兼容 | 保留worldInfoBefore/After标记 |

### 5.4 新增特性

1. **DM角色定位**：明确AI作为公正DM的角色，而非角色扮演伙伴
2. **骰子透明化**：所有检定结果必须清晰展示投骰过程
3. **行动选项**：每次输出必须提供明确的可选行动
4. **等待机制**：强制要求等待玩家输入
5. **按需数据表**：仅在必要时输出数值信息

## 六、使用建议

### 6.1 配套世界书

此预设需配合 `src/AIDM/世界书/` 目录下的YAML文件使用，建议导入顺序：

1. 00-核心定义（必须）
2. 02-角色系统（必须）
3. 03-战斗系统（推荐）
4. 04-检定系统（必须）
5. 05-世界系统（按需）
6. 07-变量定义（推荐）
7. 08-示例与模板（推荐）

### 6.2 调参建议

- 如果输出过于机械，可适当提高temperature至1.1
- 如果规则判定不稳定，可降低temperature至0.9
- 如果输出过长，可减少openai_max_tokens至2048

### 6.3 已知限制

1. 不适用于纯角色扮演场景
2. 不支持复杂的情感互动描写
3. 输出格式较为固定，灵活性较低

### 6.4 正则自定义

如需自定义正则美化效果，可修改 `格式正则（美化输出）` 模块中的内容：

**添加新的格式美化：**
```
<regex order=3>"/你的标记/gs":"替换内容"</regex>
```

**示例：添加分隔线**
```
<regex order=3>"/【新回合】/gs":"\\n════════════════════\\n🔄 新回合\\n════════════════════"</regex>
```

**注意：** 正则中的特殊字符需要转义：
- `\n` → `\\n`
- `\s` → `\\\\s`
- `\S` → `\\\\S`
- `/` → `\\/`
