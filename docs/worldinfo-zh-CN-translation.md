# 世界信息

**世界信息（也称为背景资料书或记忆书）是 ST 中的一种强大工具，可以在聊天中动态插入提示，以帮助指导 AI 的回复。**

通常情况下，世界信息（简称 WI）用于增强 AI 对您虚构世界细节的理解，但您也可以使用世界信息条目插入您希望插入到提示中的任何内容。

它就像一个动态字典，只有当消息文本中包含与条目相关的关键词时，才会从世界信息条目中插入相关信息。

SillyTavern 引擎启动，并将适当的背景故事无缝整合到提示中，为 AI 提供背景信息。

*需要注意的是，虽然 World Info 有助于引导 AI 生成期望的内容，但它并不保证这些内容会出现在生成的输出消息中。这取决于你的模型如何利用额外信息！*

## 专业技巧

*   World Info 引擎是一个非常强大的提示管理工具。不要只专注于添加角色背景故事，尽情尝试吧。
*   激活关键词、标题和其他不在**内容**字段中的信息不会被插入到上下文中，因此每个世界信息条目都应该有一个全面、独立的描述。
*   为了创建丰富而详细的世界传说，条目可以通过递归激活相互链接和引用。更多关于[递归](#recursive-scanning)的内容见下文。
*   SillyTavern 为插入的背景信息提供了灵活的上下文预算。为了节省提示令牌，建议保持条目内容简洁。

## 特定情境来源

除了全球性的世界信息选择器，传说书还可以分配到特定情境中——一个角色、一个身份或一个个人聊天。当相关的角色、身份或聊天处于活跃状态时，这些特定情境的来源会自动激活。

### 角色传说

可选地，World Info 文件可以分配给角色，作为与该角色在所有聊天（包括群组）中的专用背景资料来源。

每个角色只能绑定一个主要的 World Info。要完成此操作，请导航到角色管理面板，点击地球按钮，然后从下拉列表中选择 World Info 并点击“确定”。在导出角色时，此文件也将嵌入到角色卡片数据中。

要解除绑定、更改或为角色分配额外的 World Info 文件作为背景资料，请按住 Shift 键点击地球按钮，或点击“更多...”然后“链接 World Info”。请注意，只有主要的 World Info 文件会随角色一起导出。

当重命名 World Info 文件时，系统会提示您选择性地跨所有引用它的角色重新链接背景资料簿。这确保在重命名后现有的角色背景资料绑定保持完整。

#### 角色背景信息插入策略

在生成 AI 回复时，角色世界信息条目将使用以下策略之一，与全局世界信息选择器的条目进行组合：

##### 均匀排序（默认）

所有条目将根据其插入顺序进行排序，就好像它们是一个大文件的一部分，忽略来源。

##### 角色背景优先

角色世界信息条目将按插入顺序首先被包含，然后是全局世界信息条目。

##### 全局背景优先

全局世界信息条目将按插入顺序首先被包含，然后是角色世界信息条目。

### 角色传说书

可以将一个世界信息文件分配给当前活跃的[角色](/Usage/personas.md) 。当选择该角色时，该传说书的条目将始终被激活，无论当前打开的是哪个角色或聊天界面。

要为角色分配传说书，请打开**角色管理**面板，并点击 **角色传说**按钮。从下拉列表中选择一个世界信息文件。如果已有传说书绑定，点击按钮将直接打开传说书编辑器。

!!! 当传说书已绑定时，要重新打开分配弹窗，请在移动端使用 **长按**或在桌面端使用 **Shift+点击 / Alt+点击**操作角色传说按钮。!!!

### 聊天传说书

一个世界信息文件也可以分配给单个聊天。绑定到聊天的传说书条目仅在特定对话中有效，不会传递到其他使用相同角色的聊天中。

要为聊天分配传说书，请点击角色管理面板中的 **聊天传说**按钮。从下拉列表中选择一个世界信息文件。如果传说书已经绑定，点击按钮将直接打开传说书编辑器。

!!! 当传说书已经绑定时，要重新打开分配弹窗，请在移动设备上长按 **聊天传说**按钮，或在桌面设备上使用 **Shift+点击 / Alt+点击** 。!!!

## 世界信息条目

### 键

触发世界信息条目激活的关键字列表。键默认不区分大小写（这[可配置](#case-sensitive-keys) ）。

#### 作为键的正则表达式

键通过支持正则表达式，允许更灵活的匹配方式。这使得能够匹配包含可选单词或字符、空格以及正则表达式提供的所有其他功能的更动态的内容。
如果定义的键是一个有效的正则表达式（JavaScript 正则表达式风格，使用\` `/`\`作为分隔符。所有标志都允许），在检查是否应触发条目时，它将被视为正则表达式。多个正则表达式可以作为单独的键输入，并协同工作。在正则表达式中，逗号是可能的。纯文本键不支持逗号，因为它们被视为键分隔符。

高级正则表达式匹配用例的示例：
当字符执行与天气相关的动作时，应插入的条目/指令

```js
/(?:{{char}}|he|she) (?:is talking about|is noticing|is checking whether|observes) (?:the )?(rainy weather|heavy wind|it is going to rain|cloudy sky)/i
```

关于正则表达式语法和功能更多信息： [正则表达式 - JavaScript | MDN](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Regular_expressions)

##### 高级正则表达式按消息匹配

ST 在 WI 扫描缓冲区的每个聊天消息前都加上 `角色名:`，并且在 v1.12.6 之后，使用角色值 1 (`\x01`) 作为前缀。
这意味着你可以使用与该分隔字符绑定的正则表达式来匹配来自特定角色的特定输入或输出。

例如，若只想匹配说出"hello"的用户，可以使用以下正则表达式：

```js
/\x01{{user}}:[^\x01]*?hello/
```

#### 关键输入

输入关键词有两种模式，每种模式具有略微不同的界面。在⌨️ *纯文本模式* （默认）中，关键词可以作为一个逗号分隔的列表输入到一个文本字段中。也可以包含正则表达式，但它们没有特殊的突出显示。在✨ *炫彩模式*中，关键词会作为单独的元素显示，正则表达式也会相应地突出显示。该控制器支持编辑和删除关键词。模式可以通过输入控制器内的内联按钮进行切换。

### 可选过滤器

以逗号分隔的附加关键字与主键结合使用。如果没有提供参数，此标志将被忽略。支持逻辑运算符 AND ANY、NOT ANY 或 NOT ALL

1.  AND ANY = 仅当主键和任意一个可选过滤键在扫描上下文中时，才激活该条目。
2.  AND ALL = 仅当主键和所有可选过滤键都存在时，才激活该条目。
3.  NOT ANY = 仅当主键和没有任何一个可选过滤键在扫描上下文中时，才激活该条目。
4.  并非所有 = 如果所有可选过滤器都在扫描的上下文中，则防止条目因主键触发而激活。

这些键还支持 [正则表达式](#regular-expression-regex-as-keys) 。

### 条目内容

条目激活时插入到提示中的文本。

### 插入顺序

数值。如果多个条目同时被激活，它定义了条目的优先级。具有较大顺序号的条目将插入到上下文的更接近末尾的位置，因为它们对输出有更大的影响。例如，一个具有100顺序号的条目将出现在上下文中，而一个具有250顺序号的条目将出现在其后。

### 插入位置

*   **在角色定义之前：**World Info 条目插入到角色的描述和场景之前。对对话有中等程度的影响。
*   **在角色定义之后：** 世界信息条目插入在角色描述和场景之后。对对话的影响更大。
*   **在示例消息之前：** 世界信息条目被解析为示例对话块，并插入在角色卡提供的示例之前。
*   **在示例消息之后：** 世界信息条目被解析为示例对话块，并插入在角色卡提供的示例之后。
*   **作者注释顶部：** 世界信息条目插入在作者注释内容的顶部。对对话的影响取决于作者注释的位置。
*   **AN 的底部：** 世界信息条目插入在作者注释内容的底部。其影响程度取决于作者注释的位置。
*   **@ D：** 世界信息条目插入在聊天的特定深度（深度 0 为提示的底部）。
    *   ⚙️ - 作为系统角色消息
    *   👤 - 作为用户角色消息
    *   🤖 - 作为助手角色的消息
*   **输出端口：**World Info 条目不会自动注入。相反，其内容存储在命名的输出端口下，您可以通过调用 [`{{outlet::Name}}` 宏](#outlet-name)来决定它在提示中的确切位置。

示例消息条目将根据提示构建设置进行格式化：指令模式或聊天完成提示管理器。它们还遵循示例消息行为规则：在完整上下文中逐渐被推出、始终保留或完全禁用。

如果您的作者笔记被禁用（插入频率=0），A/N 位置中的 World Info 条目将被忽略！

### 输出端口名称

当选择**输出端口**插入位置时，将出现一个额外的**输出端口名称**字段供输入。在此处提供的名称将条目分组，并定义您将用于手动将它们拉入提示的标记。

在[提示管理器](./Prompts/prompt-manager.md)或[高级格式化](./Prompts/advancedformatting.md)提示字段中使用 {{outlet::YourName}} 宏。当构建提示时，该宏将被所有具有相同输出端口名称的 World Info 条目的组合内容替换，内容之间用换行符分隔，并按其[插入顺序](#insertion-order)值排序。

如果一个输出端口条目缺少名称，它将在生成过程中被跳过，因此请确保填写该字段。输出端口名称支持基于您已使用过的名称的自动完成，以便轻松重用一致的标签。

#### 限制和注意事项

*   将输出宏放在世界信息条目内是不支持的，并且无法工作。这与世界信息的评估顺序冲突，并可能导致无限循环。
*   嵌套输出是不支持的。你不能将输出宏放在另一个输出的内容内。与上述情况相同，这可能导致无限循环。
*   角色卡字段（描述、性格、场景等）不能扩展输出。这些字段在早期解析，因此可以作为[附加匹配源](#additional-matching-sources)用于世界信息触发器，这意味着在处理其文本时输出不可用。如果你需要在提示正文内放置输出内容，请使用另一个宏感知字段。
*   作者注释编辑器也无法解析出口。要在作者注释周围放置出口内容，应将条目分配给 **AN 顶部** 或 **AN 底部** 插入位置，而不是依赖宏。
*   出口名称区分大小写。宏 `{{outlet::}}` 必须与条目的 **出口名称** 完全相同的大小写，否则将不返回内容。
*   调用宏时，出口名称中的前导或尾随空格将被忽略，因此使用额外空格保存的名称将无法匹配。避免填充名称，以便它们能够正确解析。
*   没有分配内容的出口宏将被替换为空字符串。

### 条目标题/备注

一个方便您标记条目的文本字段，AI 或任何扳机逻辑都不会使用它。

如果为空，可以通过点击“填充空白备注”按钮，使用条目的第一个键进行回填。

### 策略

1.  🔵 (蓝色圆圈) = 该条目将始终出现在提示中。
2.  🟢 (绿色圆圈) = 该条目仅在存在关键词时才会被触发。
3.  🔗 (链式链接) = 该条目允许通过嵌入相似性插入。

每个条目都有一个开关，允许您启用或禁用该条目。

### 概率（触发百分比）

这个值充当一个额外的过滤器，当条目以任何方式（常量、主键、递归）被激活时，它会增加条目不被插入的概率。

1.  概率为100表示条目将在每次激活时被插入。
2.  概率为50表示条目将以1:1的概率被插入。
3.  概率为0表示该条目将不会被插入（实际上相当于禁用它）。

使用此功能可以在您的聊天中创建随机事件。例如，每条消息都有1%的几率唤醒一位古老之神，如果消息中提到了它的名字。

### 包含组

包含组控制当同时触发多个具有相同组标签的条目时，如何选择条目。如果多个具有相同组标签的条目被激活，只有一个将被插入到提示中。

默认情况下，所选条目会根据其组权重（默认为100分）随机选择——数字越高，被选中的概率就越高。这允许在触发的条目中进行随机选择，为互动增添惊喜和多样性。

如果条目被定义为逗号分隔的列表，一个条目可以属于多个包含组。将应用上述相同逻辑。如果该条目被触发，它将 *禁用* 所有属于其任何组的其他条目。因此，如果任何组被激活，该条目将不会被激活。

### 优先包含

为了更精确地控制通过 [包含组](/Usage/worldinfo.md#inclusion-group) 激活的条目，您可以使用“优先包含”设置。此选项允许您确定性地选择条目，而不是随机投掷组权重的机会。

如果多个具有相同组标签且此设置已开启的条目被激活，则具有最高“Order”值的条目将被选中。这对于通过包含组创建回退序列很有用。例如，可以优先考虑低深度条目并给予更多强调，或者在两个条目都有效时选择一个特定的场景设置指令而不是另一个。

### 使用组评分

当此设置全局启用或针对每个条目启用时，激活的条目键的数量决定了组获胜者的选择。只有具有最高键匹配数量的组子集将保留供组权重或包含优先级激活——其余的将被停用并从组中移除。

使用此设置以在大型组中为单个条目提供更多特异性。例如，它们可以有一个公共键和一个特定键。当未提供特定键时，将插入一个随机条目，反之亦然。

主键的得分计算逻辑是1次匹配=1分。

对于次键，交互取决于所选择的筛选逻辑：

1.  AND ANY：1 次次键匹配=1 分。
2.  AND ALL：如果所有次键都匹配，则每个次键得 1 分。
3.  NOT ANY 和 NOT ALL：无变化。

示例：

*   条目 1。关键词：song，sing，Black Cat。分组：songs
*   条目 2。关键词：song，sing，Ghosts。分组：songs

输入 `sing me a song` 可以激活任意条目（均激活 2 个键），但 `sing me a song about Ghosts` 仅会激活条目 2（激活 3 个键）。

### 自动化 ID

允许将 World Info 条目与快速回复扩展中的 [STscripts](/For_Contributors/st-script.md) 集成。如果快速回复指令和 WI 条目具有相同的自动化 ID，当激活具有匹配 ID 的条目时，指令将自动执行。

自动化将按照触发顺序执行，遵循您指定的排序策略，结合 [角色背景插入策略](#character-lore-insertion-strategy) 与 '优先级' 排序。这导致 [蓝圈](#strategy) 条目首先被处理，然后按指定 '顺序' 处理其他条目。递归触发的条目将在相同顺序中稍后处理。

脚本命令如果多个具有相同自动化 ID 的条目被激活，将只运行一次。

### 角色过滤器

一个角色名称列表，该条目可以为此列表中的角色被激活。如果此列表不为空，条目将只对列表中的角色名称被激活。当选择一个标签时，条目将只对具有该特定标签的角色被激活。

"排除"模式反转过滤器，这意味着条目将对所有角色被激活，除了那些被添加到列表中的角色或具有所选标签的角色。

### 触发器

此 World Info 条目可被激活的生成类型。如果未选择任何类型，该条目可对所有生成类型生效。如果选择了一个或多个类型，该条目将仅对所选特定生成类型生效：

*   **普通：** 常规消息生成请求。
*   **继续：** 当按下继续按钮时。
*   **模拟:** 当模拟按钮被按下时。
*   **滑动:** 当通过滑动触发生成时。
*   **重新生成:** 在单人聊天中按下重新生成按钮时。
*   **安静:** 背景生成请求，通常由 [扩展](/extensions/index.md) 或 [STscript](/For_Contributors/st-script.md) 命令触发。

!!! "再生"扳机在群聊中不可用，因为它使用不同的再生逻辑：所有来自最后回复的消息都会被删除，并且根据所选的 [群聊回复策略](/Usage/Characters/groupchats.md#reply-order-strategies)使用"正常"的生成类型进行排队。 !!!

### 附加匹配源

默认情况下，World Info 条目仅与当前对话内容进行匹配。这些选项允许您将条目与聊天中未显示的不同角色信息进行匹配，甚至还可以与角色设定信息进行匹配。当您希望拥有一个广泛适用于多个角色的条目列表，但不想管理大量的标签列表，或者不想每次创建新角色时都要更新角色过滤器列表时，这非常有用。这还允许您根据当前激活的角色设定进行条目匹配。

*   **角色描述** ：与角色描述进行匹配。
*   **角色性格** : 与高级定义下的角色性格摘要进行匹配。
*   **场景** : 与高级定义下的角色指定场景进行匹配。
*   **人格描述** : 与当前选定的人格描述进行匹配。
*   **角色备注** : 与角色备注进行匹配，该备注可在高级定义下找到。
*   **创作者笔记** ：与字符创作者的笔记匹配，这些笔记可以在高级定义下找到。创作者的笔记通常不包括在提示中。

## 向量存储匹配

向量存储扩展通过使用最近聊天消息与 World Info 条目内容之间的相似性，提供了一种替代关键词匹配的方法。

要启用和使用此功能，需要满足以下前提条件：

1.  向量存储扩展已启用，并配置为使用可用的嵌入源之一。
2.  在向量存储扩展设置中，"为世界信息启用"复选框已被勾选。
3.  允许无密钥匹配的世界信息条目具有"向量化"（🔗）状态，或者向量存储设置中已勾选"为所有条目启用"选项。

扩展中选择的向量化模型以及"嵌入"一词背后的理论含义在此不予以涵盖。如需了解更多相关信息，请查阅 [Data Bank](/Usage/Characters/data-bank.md#vector-storage) 指南。

向量存储匹配遵循以下规则：

*   允许与向量存储匹配的最大条目数可以通过"最大条目数"设置进行调整。该数字仅设置限制，不会影响在 World Info 激活设置中设置的 token 预算。所有预算规则仍然适用。
*   此功能仅替换关键词检查。要插入条目，必须满足所有附加检查：扳机%、角色过滤器、包含组等。
*   激活设置或条目覆盖中的"扫描深度"设置不使用。相反，使用向量存储的"查询消息"值来获取要匹配的文本。这允许配置"扫描深度"设置为0，因此不会进行任何正则关键词匹配，但条目仍然可以通过向量被激活。
*   "A"状态仅是一个附加标记。条目仍会像普通、启用、非常量记录一样行为，如果设置了关键词，它会被激活。如果你希望它们仅被向量激活，请移除关键词。

!!!info 注意 由于检索质量完全取决于嵌入模型的输出，因此无法准确预测将插入哪些条目。如果你需要确定性和可预测的结果，请坚持使用关键词匹配。 !!!

## 定时效果

通常，World Info 评估是无状态的，这意味着评估结果相同，仅取决于当前的聊天上下文。然而，随着定时效果的引入，你可以创建具有激活延迟、被触发后保持激活状态或激活后无法被触发的条目。

### 定时效果规则

1.  效果的时间框架以消息为单位（而不是消息/交换的对数），0表示没有效果。
2.  效果仅在消息被激活的聊天中生效。分支继承父聊天的状态。
3.  如果聊天没有进展（例如，如果最后一条消息被滑动或删除），则活动的定时效果会被移除。
4.  对当前正在生效的词条进行任何修改会导致效果被强制移除。
5.  即使效果已激活，后续关键词触发也不会刷新效果持续时间。

### 定时效果类型

1.  粘性 - 激活后词条会持续 N 条消息。已粘性的词条在过期前会忽略后续扫描的概率检查。
2.  冷却时间 - 激活后，在接下来的 N 条消息内无法再次激活。可与粘性功能结合使用：当粘性持续时间结束时，条目进入冷却状态。
3.  延迟 - 除非在评估时刻聊天中至少有 N 条消息，否则条目无法被激活。
    *   延迟 = 0 -> 条目可以随时被激活。
    *   延迟 = 1 -> 如果聊天为空（没有问候），条目无法被激活。
    *   延迟 = 2 -> 如果聊天中为零条或仅有一条消息，则无法激活该条目等。

### 定时效果示例

条目配置：粘性 = 3，冷却时间 = 2，延迟 = 2。

```txt
Message 0: delay
Message 1: entry activated
Message 2: sticky
Message 3: sticky
Message 4: sticky
Message 5: cooldown
Message 6: cooldown
Message 7: entry can be activated again
```

## 激活设置

世界信息屏幕顶部的可折叠菜单。

### 扫描深度

> 可在条目级别上覆盖。

定义应扫描聊天历史记录中的多少条消息以查找世界信息键。

*   如果设置为0，则仅评估递归条目和作者备注。
*   如果设置为 1，则 SillyTavern 仅扫描最后一条消息。
*   2 = 两条最后消息，等等。

### 包含名称

定义是否应在扫描的文本缓冲区中将聊天参与者的姓名作为消息前缀包含。这允许激活使用姓名作为关键词的条目，而无需在消息中直接提及姓名。

下方是待扫描文本的示例，假设聊天参与者名为 Alice 和 Bob。

启用（默认）：

```txt
Alice: Hello! Good to see you.
Bob: How is the weather today?
```

禁用：

```txt
Hello! Good to see you.
How is the weather today?
```

### 上下文百分比 / 预算

**定义了 World Info 条目一次性可以使用多少个 token。** 您可以相对于 API 的最大上下文设置定义一个阈值（上下文百分比）或一个目标 token 阈值（预算）

如果预算用尽，即使提示中存在相应的键，也不会再激活更多条目。

常量条目将首先插入。然后插入具有较大序号的条目。

直接通过提及它们的键插入的条目比那些在其它条目内容中提及的条目具有更高的优先级。

### 最小激活次数

**此设置与最大递归步骤互斥。**

最小触发次数：如果设置为非零值，这将忽略"扫描深度"的限制，从最新消息开始向后搜索聊天记录中的关键词，直到触发指定数量的条目为止。这仍然会受到最大深度设置或您的总体预算上限的限制。

*由最小触发次数触发的额外扫描不会检查先前步骤中递归添加的条目。只有聊天消息和扩展提示可以触发这些额外激活。然而，由最小触发次数触发的条目可以像往常一样触发其他条目。*

### 最大深度

在使用最小触发次数设置时扫描的最大深度。

### 递归扫描

递归扫描允许条目激活其他条目或被其他条目激活，从而在不同 World Info 条目之间实现复杂的交互和依赖关系。此功能可以显著增强您创意场景的动态特性。
是否启用递归扫描可以通过全局设置 **递归扫描** 进行控制。
每个条目控制递归有三个选项可供选择：

8 **不可递归** ：当选中此复选框时，该条目不会被其他条目激活。这对于不应更改或受其他世界信息条目影响的静态信息很有用。

*   **防止进一步递归** ：选中此选项可确保一旦该条目被激活，它将不会触发任何其他条目。这有助于避免意外的激活链。
    
*   **递归延迟** ：该条目仅在递归检查期间才会被激活，这意味着它不会在初始遍历中被触发，但可以被其他已启用递归的条目激活。现在，通过为这些延迟添加了**递归级别** ，条目将按级别分组。最初，只有第一级（编号最小）会匹配。一旦没有找到匹配项，下一级将变得有资格进行匹配，重复此过程，直到检查完所有级别。这允许更精确地控制递归过程中何时以及如何显示更深层次的信息，尤其是在与 NOT ANY 或 NOT ALL 关键匹配组合使用时。
    

**条目可以通过在内容文本中提及它们的关键词来激活其他条目。**

例如，如果你的 World Info 包含两个条目：

```txt
Entry #1
Keyword: Bessie
Content: Bessie is a cow and is friends with Rufus.
```

```txt
Entry #2
Keyword: Rufus
Content: Rufus is a dog.
```

**两者**都会被拉入上下文，如果消息文本中提及**仅仅是 Bessie**。

### 最大递归步骤

**此设置与最小激活量互斥。**

设置为0时，递归嵌套仅受您的提示预算限制。设置为非零值时，限制扫描遍历的总次数至期望的最大“嵌套级别”。

示例值：

*   1实际上禁用了递归，因为检查在第一步后停止。
*   2 只能激活递归条目一次。
*   3 可以触发两次递归...

### 区分大小写的键

> 可以在条目级别被覆盖。

**要进入上下文，条目键需要与 World Info 条目中定义的大小写匹配。**

当你的键是常用词或常用词的一部分时，这很有用。

例如，当此设置激活时，键'rose'和'Rose'将根据输入被不同对待。

### 匹配整个单词

> 可在条目级别被覆盖。

如果键只包含一个单词，则只有在搜索文本中包含整个单词时才会匹配。默认启用。

例如，如果设置已启用且条目键是"king"，那么"long live the king"这样的文本会被匹配，但"it's not to my liking"则不会。

**重要：** 当与不使用空格分隔单词的语言（例如日语或中文）一起使用时，此设置可能会有负面影响。如果你用这些语言编写条目，建议禁用此设置。

### 溢出警报

如果激活的世界信息超出了分配的代币预算，则显示警报。
